terminal and console

  • terminal

    terminal可以理解为输入输出设备, 用户或者管理员通过terminal来和主机进行交互.

  • console

    控制台是特殊的terminal.

    Terminal 和 Console 的区别是什么？ - 蓝形参的回答 - 知乎 https://www.zhihu.com/question/20388511/answer/14984885

    **但是在主机的系统启动完成之前，terminal是不能连接到主机上的。**为了能记录出主机开机过程的日志，也便于在主机出故障无法启动操作系统时进行检修维护，有多了一个叫做控制台的设备。**一台主机有且只能有一个控制台。**文件主机的重要日志，比如开机关机的日志和记录，重要应用程序的日志，都会输出到控制台来。

shell

shell 是解释器, 用来解释和执行从terminal传来的命令. 比如在 terminal 里输入 ls, shell 就会去找 ls 函数然后调用他.

tty,pty,pts

  • tty

    = terminal.

    在以前通常每个terminal都有一个物理屏幕和键盘连接向主机, 但现在基本都是虚拟的了.

    linux系统中有六个虚拟的terminal, 也就是tty, 可以按 ctrl-alt-F1 到 ctrl-alt-F6 切换, 其中第一个 terminal 是图形 terminal.

  • pty

    psudoterminal interfaces. 伪终端, 它就是一个跑在 user space的程序.

    pty 由 pts 和 ptm 组成. pts 是 slave 端, ptm 是 master 端.

    master端监听显示器和键盘的输入, 然后通过 TTY driver 输入到 slave, shell 的stdin, stdout, stderr 都会设置为slave端.

    slave 端本质上是个文件, 比如上面的 /dev/pts/3.

实验内容

在 lab2 开启分页机制的基础上, lab3 实现了虚拟内存管理.

主要理解ucore是如何进行虚拟内存空间的管理.

  1. 实现Page Fault异常处理
  2. 实现页替换算法

过程概览

lab3 在 kern_init 函数中增加了三个函数

  • vmm_init

    测试了两个部分, 一是测试虚拟内存空间的管理能否正确进行, 二是测试缺页异常的实现是否正确.

  • ide_init

    对我们要用到的swap 硬盘进行初始化. 看不太懂☠️

  • swap_init

    做了三件事.

    1. 获得磁盘的最大存储物理页数.
    2. 初始化页替换机制
    3. 测试页替换机制

接下来对vmm_init 和 swap_init 进行进一步分析.

lab3 中使用了一个两级的结构体vma_struct 和 mm_struct 来管理一个虚拟内存空间. 两者的定义在vmm.h 中.

前者维护虚拟内存空间的一段连续的合法区域.

然后check_vma_struct() 测试了以下这些函数.

代码都在vmm.c 中, 比较简单就不逐一解释了.

  • mm_create

  • vma_create

测试缺页异常

在这里我们实现练习1.

    完成do_pgfault（mm/vmm.c）函数，给未被映射的地址映射上物理页。设置访问权限的时候需要参考页面所在 VMA的权限,同时需要注意映射物理页时需要操作内存控制结构所指定的页表,而不是内核的页表.

  • 注意

    在LAB3 EXERCISE1处填写代码.

    执行make qemu后, 如果通过check_pgfault函数的测试后，会有“check_pgfault() succeeded!”的输出，表示练习1基本正确.

产生缺页异常的三个原因.

  1. 目标页帧不存在

    1. 虚拟地址对应的页表项全为0. 可能未和物理地址建立过映射或者映射被销毁.

    2. 找不到对应的页表项.

    3. 非法的虚拟地址.

  2. 目标页帧存在, 但不满足访问权限.

  3. 目标页帧存在, 满足访问权限, 但不在内存中

    页表项非空, 但 Present 位为0, 即可能在swap 分区中或者在磁盘中

和其他异常相比缺页异常有两个特别的地方

  1. CR2 寄存器

    CR2 寄存器中会存放产生缺页异常的线性地址, 这样在异常处理的时候就可以根据该线性地址找到对应的页表项.

  2. error code

    • P flag(bit 0)

      目标页帧的Present 位

    • W/R flag(bit 1)

      产生异常的操作是读还是写. 0 表示读, 1表示写

    • U/S flag(bit 2)

      是从内核态还是用户态产生的异常

我们需要实现 do_pgfault 函数来处理以上三种缺页异常的产生.

在实现的时候我们先处理第二种异常, 即权限错误. 再处理其他两种异常.

因为第一种异常只会让P flag = 0, 所以是可以兼容到第二种异常处理中的.

    最低的三位 P/RW/US 可以用来辅助缺页异常的.

    P Present位用来判断是否在内存中, 不在的话要进行页替换.

    D Dirty位表示是否被写过, 如果被写过换出到磁盘时要再写回磁盘.

    A Access位表示是否被访问过. 页替换机制一般倾向把没被访问过的换回磁盘.

页替换机制

页替换机制由两部分组成, 一个是页替换算法, 一个是交换内存和磁盘的外部页面调度程序.

ucore 为前一部分提供了很好的模块化封装, 我们可以用它的封装来简洁清晰的实现页替换算法.

页替换算法

ucore使用一个结构体 swap_manager 来管理所有和页替换算法有关的部分.

我们可以给该结构体绑定不同的页替换算法. 在 lab3 中使用 FIFO(先进先出) 算法. 也是借助之前一直在用的双向链表结构体来实现队列.

只需要关心两个部分. 一个是换入一个是换出

  • _fifo_map_swappable

    该函数把页面设置为可以被替换出去的状态. 通过把它加入到 FIFO 队列中来实现.

  • _fifo_swap_out_victim

    把页面从 FIFO 队列中踢出.

以上函数都需要传入虚拟内存控制器mm, 说明 ucore 执行的是局部的页面置换策略. 即每个进程只把它使用的最老的页面换出去.

页面调度程序

页替换算法的换入换出都是逻辑上的换入换出, 还需要进一步实现内存中的物理页和磁盘中的物理页的交换.

  • swap_in

    根据虚拟地址对应的页表项, 再得到它在磁盘里的内容, 然后载入到它对应的物理页中.

    但实际上这里的 ptr_result 并不是页表项对应的物理页. ucore的做法是把磁盘内容载入到任意一个空闲页中, 然后修改页表项指向该空闲页.

    在上面的 do_pgfault 中可以看到实现.

  • swap_out

    从 fifo 队列取n页出来到磁盘中.

    1. 取出物理页.
    2. 写入磁盘.
    3. 释放物理页, 取消tlb缓存.

    我们知道swap_in 的调用是在do_pgfault 中, 那么swap_out 什么时候会被调用呢. 答案是在分配物理页的 alloc_pages 函数中.

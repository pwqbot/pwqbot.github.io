DES 加密算法

概述

  • 特点

    1. 分组加密算法. 每 64bit 为一组
    2. 对称加密算法
    3. 64bit 的密钥长度, 但每第 8 位为奇偶校验位. 所以有效长度为 56bit
    4. 密钥可为任意的 56bit 数, 但存在弱密钥
    5. 采用混乱和扩散的组合，每个组合先替代后置换，共16轮
    6. 只使用了标准的算术和逻辑运算，易于实现

    当然因为是对称算法, 所以需要保证密钥传输的机密性..

  • 一般概述:

    1. 输入 64 bit 的待加密文本和 长度为 64bit, 有效位为 56bit 的密钥.
    2. IPIPIP 置换 64 bit 的待加密文本
    3. 16 轮迭代加密.
    4. 交换左右 32 bit
    5. 逆置换 IP−1IP^{-1}IP−1
    6. 输出 64bit 密文

加密过程

总共两部分. 置换IP,IP−1IP,IP^{-1}IP,IP−1 和 16 轮的迭代. 16轮迭代是算法核心.

16轮迭代

进行16轮迭代. 每一轮迭代的输入为上一轮迭代的输出.

这里又分成两块. 一是单轮迭代的实现. 二是每一轮迭代的子密钥的生成.

子密钥KiK_{i}Ki​ 的生成

由原来的 64 bit 的密钥. 生成 16 个 48 bit 的子密钥, 用于每一轮迭代.

  1. 经过置换选择 1, 去掉校验位.
  2. 根据循环表和当前轮数, 循环左移 1 位或 2 位
  3. 压缩置换选择 2, 生成 48 bit 的子密钥.

单轮置换流程

输出的 右32bit 为原来的 左32bit.

输出的 左32bit经过:

  1. 扩展运算 E. 从 32 bit 扩展到 48 bit

    一个简单的填充操作

    扩展置换E

  2. 和 48 bit 的子密钥KiK_{i}Ki​ 异或

  3. 压缩运算 S. 从 48 bit 重新压缩回 32 bit

    把48 bit 以每 6 bit 为一组, 分成 8 组. 第 iii 组经过 SiS_{i}Si​ 盒, 压缩成 4 bit. 最后组合成 32 bit.

    ​

    压缩运算S

    每个 S 盒是一个 4(2bit) 行 16(4bit) 的矩阵, 每行是 0-15 的一个排列. 以输入的 6 bit 的中间 4bit 作为列选择子, 第1和 第6 bit 合起来作为行选择子.

    选择出来的值即为压缩运算的结果.

    选择过程

    DES中其它算法都是线性的，而S-盒运算则是非线性的, 所以S盒是算法的关键所在, 其决定了整个算法的安全强度

    构造原则:

    • S盒的每一行是整数0，…，15的一个置换
    • 没有一个S盒是它输入变量的线性函数
    • 改变S盒的一个输入位至少要引起两位的输出改变
    • 对任何一个 S盒和任何一个输入 X , 输出 S(X) 和 S(X$\oplus$001100) 至少有两个比特不同
    • 对任何一个 S盒, 和任何一个输入对 e,f 属于{0,1}, S(X) ≠\neq= S(X$\oplus$11ef00)
    • 对任何一个S盒,如果固定一个输入比特,来看一个固定输出比特的值, 这个输出比特为 0 的输入数目将接近于这个输出比特为1的输入数目

  4. 置换 P

    简单的位置置换 .目的是提供雪崩效应, 明文或密钥的一点小的变动都引起密文的较大变化

    P置换矩阵

  6. 和原来的左 32 bit 异或

工作模式

  1. 电子密码本 ECB (electronic codebook mode)
  2. 密码分组链接 CBC (cipher block chaining)
  3. 密码反馈 CFB (cipher feedback)
  4. 输出反馈 OFB (output feedback)

ECB

最简单的分组密码方式. 每段明文独立加密解密.

Ci=Ek(Pi)Pi=Dk(Ci)C_{i} = E_{k}(P_i) \\ P_{i} = D_{k}(C_{i}) Ci​=Ek​(Pi​)Pi​=Dk​(Ci​)

  • 优点:

    1. 简单有效
    2. 可以并行实现
    3. 误差传递: 某一密文块损坏, 仅对应的明文块损坏. 错误在分组之间独立.

  • 缺点

    1. 不能隐藏明文的模式信息. 比如加密一张图片, 可以看出大致轮廓

    2. 因为不管什么时候, 使用同样的明文和同样的密钥, 就能得到同样的密文, 所以容易受到主动攻击.

      设某个协议支持银行间的电汇，电汇包含五个字段，分别有五个分组

      首先一个前提是我们能监听，拦截并更改这个电汇的密文，

      然后我们可以先开个账户，进行若干次试验，每一次变更一个参数，比如发送账号，转账金额，一组密码的位数等，以此来确定每一个分组的意义。

      等我们大概知道五个分组的意义后，我们就能进行替换攻击。比如银行A的某个账户向银行B的某个账户转账，我我们将其拦截，把接受账号密文代换成事先存好的我们账号的密文，则银行用key解密后，就会把钱都转到我们的账户下。值得一提的是，整个过程我们并没有攻击分组密码本身，所以尽管加密方案本身再安全也抵御不了这种攻击；我们甚至也不知道真正明文的内容，所以实现代换攻击是破坏了消息的完整性。

      密码学学习笔记：分组密码操作模式 - 蚁景科技的文章 - 知乎 https://zhuanlan.zhihu.com/p/130732260

CBC

把上一个输出密文和下一个输入密文进行异或之后再加密. 第一组密文受初始向量 IV 影响.

Ci=Ek(Pi⊕Ci−1)Pi=Dk(Ci)⊕Ci−1C_{i} = E_{k}(P_{i} \oplus C_{i - 1}) \\ P_{i} = D_{k}(C_{i}) \oplus C_{i-1} Ci​=Ek​(Pi​⊕Ci−1​)Pi​=Dk​(Ci​)⊕Ci−1​

  • 优点
    1. 能够隐藏明文的模式信息
    2. 选择不同的初始话向量, 可以使得相同的明文加密得到不同的密文
    3. 主动攻击不容易.
    4. 解密可以并行
  • 缺点
    1. 加密不能并行实现
    2. 某个密文块CiC_{i}Ci​损坏, 会影响 PiP_{i}Pi​ 和 Pi+1P_{i + 1}Pi+1​.

CFB

主要是加了一个移位寄存器, 其余的和CBC其实差不多.

用 head(n,x)head(n,x)head(n,x) 表示 nnn 的二进制的高xxx 位.

Si=(Si−1≪j)∣Ci−1;Ci=head(Ek(Si),x)⊕PiSi=(Si−1≪j)∣Ci−1;Pi=Ci⊕head(Ek(Si,x))S_{i} = (S_{i - 1} \ll j) \mid C_{i - 1}; \quad C_{i} = head(E_{k}(S_{i}),x) \oplus P_{i} \\ S_{i} = (S_{i-1} \ll j) \mid C_{i - 1}; \quad P_{i} = C_{i} \oplus head(E_{k}(S_{i},x)) Si​=(Si−1​≪j)∣Ci−1​;Ci​=head(Ek​(Si​),x)⊕Pi​Si​=(Si−1​≪j)∣Ci−1​;Pi​=Ci​⊕head(Ek​(Si​,x))

每一次的输出CiC_{i}Ci​ 反馈到了 Si+1S_{i+1}Si+1​, 而PiP_{i}Pi​ 只是进行一个简单的异或.

注意加解密双方需要共同的移位寄存器的初始值.

对称密码算法中在使用同一密钥时，不应当两次使用同一个 IV 加密数据.

  1. 使用 OFB 工作模式或使用流密码（如 RC4）加密时，如果使用相同的IV值和密钥来加密两条消息，攻击者知道了加密后的两个消息之一的明文，就可以对另外一个消息包进行解密；

  2. 对于CFB工作模式而言，如果使用相同的IV值和密钥来加密两条消息，攻击者知道了加密后的两个消息包之一的明文，则另一个消息包的首个明文块信息泄露。

    因为使用相同的 IV值和密钥, 那么这两个消息的第一个明文块异或的值就是一样的. 把他们的密文异或之后, 就得到两个明文异或的值. 知道了其中一个, 另外一个也知道了.

  • 优点

    1. 能够隐藏明文的模式信息
    2. 选择不同的初始化向量, 可以使得相同的明文加密得到不同的密文
    3. 主动攻击不容易
    4. 解密可以并行
    5. 可以实现流密码

  • 缺点

    1. 加密不能并行实现
    2. 某个密文块CiC_{i}Ci​损坏, 会影响后面所有的明文段.

OFB

加密密钥只由最初的IV向量产生.. 明文只异或了一下生成密文.

由于明文和密文只在最终的异或过程中使用，因此可以事先对IV进行加密，最后并行的将明文或密文进行并行的异或处理。

使用 OFB 工作模式或使用流密码（如 RC4）加密时，如果使用相同的IV值和密钥来加密两条消息，攻击者知道了加密后的两个消息之一的明文，就可以对另外一个消息包进行解密；

1.(Transpose of a Directed Graph. CLRS 22.1-3) (15 points)

The transpose of a directed graph G=(V,E)G= (V, E)G=(V,E) is the graph GT=(V,ET)G^T= (V, E^T)GT=(V,ET), where ET=(v,u)∈V×V:(u,v)∈E.E^T={(v, u)∈V×V: (u, v)∈E}.ET=(v,u)∈V×V:(u,v)∈E. Thus,GTG^TGT is GGG with all its edges reversed. Describe eﬀicient algorithms for computing GTG^TGT from GGG, for both the adjacency-list and adjacency-matrix representations of GGG. Analyze the running times of your algorithms. [We are expecting: pseudo-codes and analyses of the running times.]

solution

  • adjacency-list

    every edge will be traversed once, so running times is O(∣V∣+∣E∣)O(|V| + |E|)O(∣V∣+∣E∣).

  • adjacency-matrix

    we have a outer loop of O(∣V∣)O(|V|)O(∣V∣) and inner loop of O(∣V∣)O(|V|)O(∣V∣), so the running times is O(∣V∣2)O(|V|^2)O(∣V∣2).

2. (Cubic of Directed Graph. CLRS 22.1-5) (15 points)

The cubic of a directed graph G=(V,E)G= (V, E)G=(V,E) is the graph G3=(V,E3)G^3= (V, E^3)G3=(V,E3) such that (u,v)∈E3(u, v)∈E^3(u,v)∈E3 if and only GGG contains a path with at most three edges between uuu and vvv. Describe eﬀicient algorithms for computing G3G^3G3 from GGG for both the adjacency-list and adjacency-matrix representations of GGG. Analyze the running times of your algorithms. [We are expecting: pseudo-codes and analyses of the running times.]

solution

  • adjacency-list

    the time complexity is …. O(∣E∣3)O(|E|^3)O(∣E∣3) .

  • adjacency-matrix

    let mmm be the adjacency-matrix, let m2=m×mm^2 = m \times mm2=m×m, according to the definition of matrix, if m2[i][j]=1m^2[i][j] = 1m2[i][j]=1, means there exists m[i][k]=1m[i][k] = 1m[i][k]=1 and m[k][j]=1m[k][j] = 1m[k][j]=1, which means there exists a two-edges path from iii to jjj.

    so m+m2+m3m + m^2 + m^3m+m2+m3 contains every at most three-edges path from iii to jjj, which is G3G^3G3.

    the running times depends on the time complexity of matrix multiplcation, which is normally O(∣V∣3)O(|V|^3)O(∣V∣3)​

3.(Universal Sink. CLRS 22.1-6) (10 points)

Most graph algorithms that take an adjacency-matrix representation as input require time Ω(V2)Ω(V^2)Ω(V2), but there are some exceptions. Show how to determine whether a directed graph GGG contains a universal sink—a vertex with in-degree ∣V∣−1|V|−1∣V∣−1 and out-degree 000—in time O(V)O(V)O(V), given an adjacency matrix for GGG. [We are expecting: a pseudo-code of your algorithm and an analyze of the correctness of your algorithm.]

solution

firstly, a graph GGG must contains at most one universal sink.

and the adjacency-matrix must be like

that is, if the universal sink is kkk, the kthkthkth row of matrix is all zeros, and the kthkthkth col of matrix is all 1a except for the kthkthkth row.

If we traverse form upper-left corner, and go down when meeting 1, go right when meeting 0, we will arrive the universal sink’s row.

but this is only a necessary condition. so we must check the row to see if it is all zeros.

For example,suppose that 111 U.S. dollar buys 494949 Indian rupees, 111 Indian rupee buys 222 Japanese yen, and 111 Japanese yen buys 0:01070:01070:0107 U.S. dollars.   • a.

    • we can apply logloglog function into each currency.

    R[i1,i2]⋅R[i2,i3]…R[ik−1,ik]⋅R[ik,i1]>1⇔log(R[i1,i2])+log(R[i2,i3])+⋯+log(R[ik,i1])>0⇔−log(R[i1,i2])−log(R[i2,i3])−⋯−log(R[ik,i1])<0R[i_1, i_2]·R[i_2, i_3]\dots R[i_{k−1}, i_k]·R[i_k, i_1]>1 \\ \Leftrightarrow \\ log(R[i_1,i_2]) + log(R[i_2, i_3]) + \dots + log(R[i_k,i_1]) > 0 \\ \Leftrightarrow \\ -log(R[i_1,i_2]) - log(R[i_2, i_3]) - \dots - log(R[i_k,i_1]) < 0 R[i1​,i2​]⋅R[i2​,i3​]…R[ik−1​,ik​]⋅R[ik​,i1​]>1⇔log(R[i1​,i2​])+log(R[i2​,i3​])+⋯+log(R[ik​,i1​])>0⇔−log(R[i1​,i2​])−log(R[i2​,i3​])−⋯−log(R[ik​,i1​])<0

    So the question equals to find a negative loop in the Graph, which can be solved by Bellman-Ford algorithm.

    time complexity is O(∣V∣∣E∣)O(|V||E|)O(∣V∣∣E∣)

    to output this negative loop,

    in the last relax operation of Bellman-Ford, if we find an edge (u,v)(u,v)(u,v) such that

1.2 Eight Great IDeas in Computer Architecture

八个计算机结构设计思想

  1. Moore’s Law

    价格不变时, 每18-24个月, 集成电路可容纳的元器件的数目会增加一倍

  2. Use Abstraction to Simplify Design

    使用抽象来隐藏底层复杂的结构和功能.

  3. Make the Common Case Fast

    让最重要的部分更快

  4. Performance via Parallelism

    并行计算

  5. Performance via Pipelining

    流水线工作

  6. Performance via Prediction

    预测你的性能

  7. Hierarchy of Memories

    存储的层级结构. 顶层是速度快, 小的, 昂贵的内存, 底层是速度慢, 大的, 便宜的内存

  8. Dependability via Redundancy

    用即使冗余的成分来增加程序的可靠性.

1.6 Performance

衡量指标

在衡量Performance的时候有很多指标. 对于计算机一般使用两种. 一是 throughput, 二是 response time.

  • throughput

    total amount of work done in a given time.

  • response time

    完成一个给定任务的时间.

接下来主要研究response time.

首先区别两个概念, elapsed time 和 cpu time.

  • elapsed time

    执行完一个任务的总时间, 包括io的等待时间等等.

  • cpu time

    单纯在cpu上执行的时间, 不包括io.

在cpu time 中又细分为 system time 和 user cpu time.

  • system time

    在操作系统中花的时间

  • user cpu time

    在用户态中花的时间. 即程序本身的执行时间.

1.7 The Power Wall

随着时钟频率的提高, CPU 消耗的功率也随之提高

  • dynamic energy

    要在提高频率的同时降低功耗, 通过降低电压来实现.

  • static energy

    即使芯片不工作, 也会存在电压泄露导致功率消耗. 电压越低, 晶体管数量越多, 泄露越严重.

1.8 Pitfall

  1. Expecting the improvement of one aspect of a computer to increase overall performance by an amount proportional to the size of the improvement.

    根据Amdahl’s Law 公式

    提升系统某部分的性能, 对整体性能的提升未必有想象的那么大.

  2. Computers at low utilization use little power

  3. Designing for performance and designing for energy efficiency are unrelated goals.

    性能提升能减少运行时间, 进而节约能源.

  4. Using a subset of the performance equation as a performance metric.

    评估性能请使用完整的公式.

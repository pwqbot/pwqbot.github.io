树的重心

  • 定义

    遍历无根树上的每一个点, 计算以它为根时最大子树的结点数. 使得这个结点数最小的点就是树的重心.

  • 性质

    1. 重心 ⇔\Leftrightarrow⇔ 最大子树的结点数 ≤\le≤ 树的大小的一半.

      →\rightarrow→ 如果最大子树的结点数大于树的大小的一半. 以这个子树的根结点为整棵树的根. 就得到一个更好的重心.

      ←\leftarrow← 假设此时根节点uuu 不是重心, 那么重心只有可能在最大的子树中. 很显然此时包含uuu 的那个子树大小超过树的大小的一半.

    2. 树的重心至多有两个, 且这两个重心相邻.

      首先证明这两个重心相邻. 如果不相邻, 如图设重心分别为 u, v.

      那么 uuu 的最大子树一定是包含 vvv 的那个. 如果不是 vvv 的那边. 那么 vvv 的包含uuu 的那个子树就会比uuu 的最大子树更大.

      同理 vvv 的最大子树一定是包含uuu 的那个.

      这时候取u,vu,vu,v 上的路径某点 ttt. 那么可以看到 ttt 的最大子树显然是u,vu,vu,v 的最大子树的子集. 所以 ttt 才是重心.

      所以u,vu,vu,v 一定只能相邻.

      因为一棵树上, 最多有两个点相邻 , 所以重心也最多只有两个.

      并且从上面的推导可以看出, 这两个重心, 在对方为根时, 自己所在的子树大小恰好为 树的大小的一半. 也就是说只有树的结点数为偶数的时候才可能出现两个重心.

    3. 树上所有点到某个点的距离和中, 到重心的距离和是最小的.

      想不出很严格的证明.

      假设当前选择了 uuu, 那么要选择一个相邻的更优点 vvv 的话.

      距离和减少 vvv 的子树大小, 增加整棵树除去 vvv 的大小.

      所以 vvv 这颗子树的大小≥\ge≥ 整棵树的大小的一半.

      直到走到重心, 此时达到局部最优点. 不知道怎么证这也是全局最优点.

    4. 一棵树增加或删除一个结点, 重心最多移动一个节点.

      也比较显然. 比如增加一个结点, 若重心移动, 则说明增加到的这颗子树的大小大于整棵树的一半了. 那么移过去一个点就ok了.

    5. 连接两棵树, 那么新的重心就在这两颗树的重心的连线上.

      懒得讲了

点分治

目前只做到统计路径问题可以用到. 实际上就是一种暴力.

把路径分类讨论.

  1. 经过当前根结点的

    1. 终止在根节点的
    2. 不终止在根节点的.

    暴力遍历每个子树即可得到. 一般要把O(n2)O(n^2)O(n2) 的暴力降到 O(nlog(n))O(nlog(n))O(nlog(n))

  2. 不经过当前根节点的.

    重心递归子树. 最多递归O(log(n))O(log(n))O(log(n)) 次

  • 容斥

    题目分为两种. 一种是统计量具有对称性, 一种是不具有对称性.

    对称性是指统计子树(u,v)(u,v)(u,v) 和子树 (v,u)(v,u)(v,u) 的结果是一样的.

    1. 具有对称性的, 比如路径权值和. 这时候可以一边遍历一边统计.

      就像要得到一个数组元素的两两乘积, 可以从前往后遍历, 然后再从当前往回计算就行了.

      这样每次递归只要O(n)O(n)O(n) 的时间.

    2. 不具有对称性的, 则需要获得所有子树的信息之后再做处理.

      这样通常会把同一个子树的两条路径给合并计算了. 那么就需要在计算的时候再把它减掉.

Prufer序列

每次找到编号最小的叶节点, 删除它, 把它的父节点加入prufer序列中.

对一个有n个结点的有编号树, 其prufer序列的大小为 n - 2.

为什么长度是 n - 2呢, 因为最后剩下两个点的时候, 一定有一个是n, 于是一定是 prufer[n−1]=nprufer[n - 1] = nprufer[n−1]=n. 于是n−1n - 1n−1 可以不考虑.

无根树转prufer序列

每次选择编号最小的叶结点. 把他的父亲加入perfer序列中. 删除这个结点.

直到剩下两个结点. 编号为n的结点一定被剩下.

算法

维护一个指针p, 指向编号最小的叶结点.

删除p, 检测此时是否产生叶结点.

如果产生新的叶结点, 设编号为x.

  1. 若x>p, 则继续.
  2. 若x<p, 删除x, 检测是否产生新的叶结点.

增加p到下一个叶结点.

为什么正确呢, 因为删除p后, 只有新产生的那个叶结点的编号是有可能小于p的, 之前就存在的叶子结点编号肯定是大于p的.

转为有根树

prufer序列是对一个有编号的无根树而言的.

注意到 n 一定会被剩下来. 也就是他不会被删掉. 所以可以转换成n为根的有根树, 这样可以求出每个节点的父亲节点.

然后就可以方便的求出prufer序列了.

prufer序列转无根树

由prufer序列能得到每个点的度数.

DSU

和点分治旗鼓相当的暴力算法.

解决树上询问问题. 不支持修改.

比如询问树上每个子树的颜色数.

  • 暴力

    暴力dfs. 每次从某个子树退出的时候还要清空这颗子树的答案.

    时间复杂度O(n2)O(n^2)O(n2).

  • 启发式合并

    可以这么做. 先和上面一样暴力计算轻儿子.

    然后计算重儿子. 然后再暴力把轻儿子合并到重儿子上面... 这样就能得到答案了.

  • 时间复杂度

    用到了轻重链剖分的性质: 任何一个点到根的路径上轻边的个数最多有log(n)log(n)log(n) 个. 这是因为走一次轻边所在树的大小至少减半.

    然后对于每个结点, 首先不管他上面有重边还是轻边, 他至少都会被遍历一次. 然后他到根的路径上每有一个轻边, 就会被多遍历一次. 于是每个点被遍历到的次数是O(log(n)+1)O(log(n) + 1)O(log(n)+1).

CF Round 383(DIV.1) D. Arpa’s letter-marked tree and Mehrdad’s Dokhtar-kosh paths

https://codeforces.com/problemset/problem/741/D

  • 题意

    对每个子树问最长的路径, 使得边上的字母经过排列后可以形成回文串.

  • 分析

    融合了分治的思想. 和分治一样, 对每个子树的路径, 先算他下面的, 然后再算经过他的.

    但是这题实际上要搞一个路径合并. 如果暴力合并的话, 对重儿子的那些路径, 都要一条一条更新根到重儿子的那条边, 时间复杂度肯定不行.

    在dsu on tree 中必须是, 一条轻边才可以对下面的每个结点多访问一次. 重边不能多访问, 这样才能保证复杂度.

    这道题我们实际上只需要关心路径上单词个数的奇偶性, 而不需要关心它的具体个数.

    用二进制表示一条路径上, 每个单词数量的奇偶性. 即第 i 位为 1 表示这个单词是奇数, 为0表示这个单词是偶数.

    那么两条路径可不可以合并, 就是看他们异或之后二进制里 1 的个数小于等于 1.

    而对于两条路径的异或, 他们重叠部分的贡献是没用的. 于是可以先预处理每个点到根的路径的二进制值.

    然后为了O(1)O(1)O(1) 查询, 得用一个桶存一下已经出现过的二进制值.

    然后每次查询的时候 x⊕2ix \oplus 2^{i}x⊕2i 看一下在不在桶内即可.

    时间复杂度 O(22nlogn)O(22nlogn)O(22nlogn).

  • 总结

    这道题需要对dsu on tree的复杂度有很好的理解.

    1. 重边不能多遍历, 轻边可以暴力往下遍历合并贡献.
    2. 每个点的修改或是查询必须是 O(1)O(1)O(1) 或者 O(log(n))O(log(n))O(log(n)) 的.

    而且也看出来 dsu on tree 处理树上路径是很有局限的, 对每个点只能保存到根的权值. 这样重儿子往上合并的时候才不用更新..

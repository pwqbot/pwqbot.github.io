扩展KMP

这里假设数组下标[0,n−1][0,n - 1][0,n−1].

对字符串sss, 定义z[i]z[i]z[i] 为从 iii 开始为sss 前缀的最长字符串长度.

  • 暴力求法.

    O2(n)O^2(n)O2(n) 循环.

  • 优化

    假设当前遍历到iii, 记录一下[0,i−1][0,i - 1][0,i−1] 位置上的匹配中, 最右边的那个匹配[l,r][l,r][l,r].

    那么显然l<il < il<i. 这时候有两种情况

    1. i>ri > ri>r

      暴力计算, 注意可能更新rrr.

    2. i<=ri <= ri<=r

      这时候可以知道 [i,r][i,r][i,r] 和[i−L,R−L][i - L, R-L][i−L,R−L] 这一段是相同的.

      于是位置 iii 的匹配就可以借用位置i−Li - Li−L 的匹配.

      所以然后对于R−LR- LR−L 和 RRR 之后的字符则不一定了, 于是先设置成

      z[i]=min(R−i+1,z[i−L])z[i] = min(R - i + 1, z[i - L]) z[i]=min(R−i+1,z[i−L])

      然后再从R+1R + 1R+1 暴力计算.

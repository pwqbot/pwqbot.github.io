后缀数组

定义

sufisuf_isufi​ 表示 s[i,i+1,…,n]s[i,i+1, \dots,n]s[i,i+1,…,n].

sa[i] 表示所有后缀中字典序第 i 小的后缀.

rk[i] 表示第 i 个后缀的字典序.

sa 和 rk 可以互相推导, 两者是等价的.

求后缀数组

对于长度为 nnn 的字符串, 它的后缀有 nnn 个. 如果朴素排序的话, 两个字符串的比较需要 O(n)O(n)O(n), 最后的排序时间就是 O(n2logn)O(n^{2}logn)O(n2logn) .

  • O(nlog2n)O(nlog^{2}n)O(nlog2n) 倍增算法

    思想也很简单. 第一次排序所有长度为 111 的子串， 然后排序长度为 222, 长度为 444 .

    要排序长度为 2x2x2x 的子串的时候, 因为长度为 xxx 的子串的字典序都知道了.

    于是把前半个长度 xxx 的子串作为第一关键字, 后半个长度 xxx 的子串作为第二关键字排序即可. 利用了后缀是连续的以及字典序的性质.

    每次比较 nlog(n)nlog(n)nlog(n) , 总共比较 log(n)log(n)log(n) 次. 所以总的复杂度是O(nlog2)O(nlog^2)O(nlog2)

  • O(nlog(n))O(nlog(n))O(nlog(n)) 基数排序优化.

    这里的基数排序的值域就是每次的p. 每次先按第二关键字排序, 然后按第一关键字排序. 其他都一样

LCP(最长公共前缀)

在这里定义

lcp(i,j)=Lcp(sa[i],sa[j])lcp(i,j) = Lcp(sa[i], sa[j]) lcp(i,j)=Lcp(sa[i],sa[j])

表示字典序第 iii 小的后缀和字典序第 jjj 小的后缀的最长公共前缀.

若lcp(i,j)=llcp(i,j) = llcp(i,j)=l. 那么 i,ji,ji,j 之间的后缀至少都会有这个长度为 lll 的前缀, 否则字典序不可能在i,ji,ji,j 之间.

于是可以得到以下形式化的引理.

    lcp(i,k)=min(lcp(i,j),lcp(j,k)),1≤i<j<k≤nlcp(i,k) = min(lcp(i, j), lcp(j,k)), 1 \le i < j < k \le n lcp(i,k)=min(lcp(i,j),lcp(j,k)),1≤i<j<k≤n

    证明:

    1. 首先 lcp(i,k)≥min(lcp(i,j),lcp(j,k))lcp(i,k) \ge min(lcp(i,j), lcp(j,k))lcp(i,k)≥min(lcp(i,j),lcp(j,k)).

    2. 然后证明 左边$ \le$ 右边成立.

      用反证法, 假设左边 > 右边. 根据字典序可以得出矛盾.

      设min(lcp(i,j),lcp(j,k))=l1min(lcp(i,j), lcp(j,k)) = l_{1}min(lcp(i,j),lcp(j,k))=l1​. lcp(i,k)=l2>l1lcp(i,k) = l_2 > l_1lcp(i,k)=l2​>l1​.

      根据条件可得.

      sa[i][l1+1]≤sa[j][l1+1]≤sa[k][l1+1]sa[i][l_{1} + 1] \le sa[j][l_{1} + 1] \le sa[k][l_{1} + 1] sa[i][l1​+1]≤sa[j][l1​+1]≤sa[k][l1​+1]

      因为lcp(i,k)=l2>l1lcp(i,k) = l_{2} > l_{1}lcp(i,k)=l2​>l1​. 所以

    字典序保证只会出现第二种.

    根据上面的引理容易得到

    lcp(i,k)≤lcp(i,j),i<j≤klcp(i,k)≤lcp(j,k),i<j≤klcp(i,k) \le lcp(i,j), i< j \le k \\ lcp(i,k) \le lcp(j,k), i< j \le k lcp(i,k)≤lcp(i,j),i<j≤klcp(i,k)≤lcp(j,k),i<j≤k

Height数组

定义 Height[i]=lcp(i,i−1)Height[i] = lcp(i,i-1)Height[i]=lcp(i,i−1). Height(1)=0Height(1) = 0Height(1)=0.

即字典序相邻的后缀的最长公共前缀.

  • 求任意两个后缀的最长公共前缀 $ lcp(i,j)$

    用 $ l cp$定理, 加Height数组, 等价于求 min{ht[i+1,…,j]}min\{ht[i + 1, \dots,j]\}min{ht[i+1,…,j]}. 就是一个RMQ问题.

  • O(n)O(n)O(n) 计算 HeightHeightHeight 数组

    Height[rk[i]]≥Height[rk[i−1]]−1Height[rk[i]] \ge Height[rk[ i -1 ]] - 1 Height[rk[i]]≥Height[rk[i−1]]−1

    意思是第 iii 个后缀和字典序在它前一位的后缀的 Lcp , 至少是第 i−1i - 1i−1 个后缀和字典序在它前一位的后缀的 Lcp 减1.

    证明:

    1. 如果Height[rk(i−1)]≤1Height[rk(i - 1)] \le 1Height[rk(i−1)]≤1. 那么显然成立

    2. 如果Height[rk(i−1)]>1Height[rk(i - 1)] > 1Height[rk(i−1)]>1.

      si−1=aADs_{i - 1} = aADsi−1​=aAD, ssa[[rk[i−1]−1]]=aABs_{sa[[rk[i - 1] - 1]]} = aABssa[[rk[i−1]−1]]​=aAB. Height[i−1]=aAHeight[i - 1] = aAHeight[i−1]=aA.

      那么 si=ADs_{i} = ADsi​=AD, 并且 s[rk[i−1]−1]+1=ABs_{[rk[i - 1] - 1] + 1} = ABs[rk[i−1]−1]+1​=AB 一定排在AD=sa[rk[i]]AD = sa[rk[i]]AD=sa[rk[i]] 之前.

      所以 AD>=rk[sa[rk[i]−1]]>=ABAD >= rk[sa[rk[i] - 1]] >= ABAD>=rk[sa[rk[i]−1]]>=AB. 所以 sa[rk[i]−1]sa[rk[i] - 1]sa[rk[i]−1] 一定有前缀 A.

    理解一下: ADADAD 和 ABABAB 之间 比 aADaADaAD 和 aABaABaAB 之间会有更多的后缀. 因为在后者之间的后缀删掉开头一定也在前者之间.

    用这个引理可以O(n)O(n)O(n) 求出 Height 数组.

应用

  • 比较任意两个子串s[x,x+1,…,y],s[p,p+1,…,q]s[x,x+1, \dots, y], s[p, p +1, \dots, q]s[x,x+1,…,y],s[p,p+1,…,q] 的字典序

    ​ 如果直接比较 rk[x]rk[x]rk[x] 和 rk[p]rk[p]rk[p] 是不行的, 因为 rkrkrk 算的是整个后缀的字典序. 也就是说不知道大小关系是在哪里不同的.

    ​ 可以先求出Lcp(rk[x],rk[p])Lcp(rk[x], rk[p])Lcp(rk[x],rk[p]). 得到他们的最长前缀的长度 lll.

    1. l≥min(∣s1∣,∣s2∣)l \ge min(|s_1|,|s_2|)l≥min(∣s1​∣,∣s2​∣). 那么哪个长哪个字典序大.

    2. l<min(∣s1∣,∣s2∣)l < min(|s_1|, |s_2|)l<min(∣s1​∣,∣s2​∣), 那么这时候就可以比较 rk[x]rk[x]rk[x] 和 rk[p]rk[p]rk[p] 了.

  • 统计不同子串的个数.

    ​ 所有的子串, 可以看成 所有的后缀的所有前缀.

    ​ 于是可以循环后缀, 然后看前缀.

    ​ 如果按照字典序循环, 即循环后缀数组. 因为已经知道了Lcp(i,i−1)=ht[i]Lcp(i,i-1) = ht[i]Lcp(i,i−1)=ht[i]. 所以从 i−1i - 1i−1 到 iii 的时候, 公共前缀的那部分肯定就不用算了. 而且可以断言, 公共前缀之后的前缀, 在之前都未出现. 这是因为如果在之前出现的话, 就会发生

    Lcp(i,j)>Lcp(k,j)i<kLcp(i,j) > Lcp(k,j) \qquad i < k Lcp(i,j)>Lcp(k,j)i<k

    和之前的推论相矛盾. 所以公共前缀之后的前缀就是之前都没出现过的前缀.

    于是可以得到所有不同的子串个数为

洛谷 P4051 字符加密 模板题

  • 题意

    循环字符串, 找到最小的字典序.

  • 分析

    首先把给的字符串复制一遍, 这样所有连续的长度为 nnn 的子串就是要求的那些字符串.

    然后求一遍 sasasa 就行了.

    这里有个思考点. 求 sasasa 的时候每个子串后面都会多出来一块, 这会不会影响结果呢.

    其实是不会的. 因为, 如果两个子串字典序不相同, 那么他们后面加什么都是不影响它们的相对关系的.

    最后按照题目要求, 输出 sa[i]+n−1sa[i] + n - 1sa[i]+n−1 的字符就行了.

洛谷P2870 Best Cow line 贪心

  • 题意

    给一个字符串, 每次从头或者尾取一个字符出来加入新的字符串的尾部, 问所能得到的字典序最小的字符串是什么.

  • 分析

    每次肯定是前面取一段, 然后后面取一段. 这样下去.

    因为字典序就是要贪心的取, 所以如果最前面和最后面字典序不一样, 直接取字典序小的就行了.

    如果一样的话, 就要两个指针一直往前找, 直到找到不一样的.

    其实这样就能过了, 因为每次都只有一个指针会回溯.

    如果用后缀数组的话可以更简单一点. 把原字符串取反, 然后拼在后面. 这样就能求出 rk[i] 数组.

    然后也是双指针, 每次比较原串和反串开头的rk就行了.

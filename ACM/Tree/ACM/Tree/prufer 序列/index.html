Prufer序列

Prufer序列

每次找到编号最小的叶节点, 删除它, 把它的父节点加入prufer序列中.

对一个有n个结点的有编号树, 其prufer序列的大小为 n - 2.

为什么长度是 n - 2呢, 因为最后剩下两个点的时候, 一定有一个是n, 于是一定是 prufer[n−1]=nprufer[n - 1] = nprufer[n−1]=n. 于是n−1n - 1n−1 可以不考虑.

无根树转prufer序列

每次选择编号最小的叶结点. 把他的父亲加入perfer序列中. 删除这个结点.

直到剩下两个结点. 编号为n的结点一定被剩下.

算法

维护一个指针p, 指向编号最小的叶结点.

删除p, 检测此时是否产生叶结点.

如果产生新的叶结点, 设编号为x.

  1. 若x>p, 则继续.
  2. 若x<p, 删除x, 检测是否产生新的叶结点.

增加p到下一个叶结点.

为什么正确呢, 因为删除p后, 只有新产生的那个叶结点的编号是有可能小于p的, 之前就存在的叶子结点编号肯定是大于p的.

转为有根树

prufer序列是对一个有编号的无根树而言的.

注意到 n 一定会被剩下来. 也就是他不会被删掉. 所以可以转换成n为根的有根树, 这样可以求出每个节点的父亲节点.

然后就可以方便的求出prufer序列了.

想一下要是不转有根树的话要怎么做呢. 就不好做..

prufer序列转无根树

由prufer序列能得到每个点的度数.

首先没在prufer序列中出现过的, 就是叶子结点.

那么此时编号最小的叶子结点, 会和序列的第一个数连边.

然后删除这个数, 减少他的度. 同样的, 这时候他就可能成为叶子结点. 然后和构造prufer序列的时候一样, 比较一下编号就行了.
